âš ï¸ Common Mistakes to Avoid

ğŸ”§ Git Troubleshooting

ğŸš¨ "I committed to the wrong branch"

# Move recent commits to correct branch:
git checkout correct-branch
git cherry-pick <commit-hash>
git checkout wrong-branch
git reset --hard HEAD~1

ğŸ’¥ "I need to undo my last commit"

# Keep changes, undo commit:
git reset --soft HEAD~1

# Discard everything:
git reset --hard HEAD~1

ğŸ”„ "My branch is behind/ahead"

# Update your branch:
git fetch origin
git rebase origin/main

# Or merge instead:
git pull origin main

ğŸ¯ "I want to see what changed"

git diff # Unstaged changes
git diff --staged # Staged changes
git show <commit> # Specific commit
git log --stat # Files changed

ğŸ—‘ï¸ "I want to discard changes"

# Discard unstaged changes:
git checkout -- <file>
git checkout . # All files

# Unstage files:
git reset HEAD <file>

ğŸ•°ï¸ "I need to go back in time"

# Temporarily checkout old commit:
git checkout <commit-hash>

# Create branch from old commit:
git checkout -b fix-branch <commit>

âš ï¸ Emergency Commands (Use Carefully!)

git fsck --full # Check repository integrity
git gc # Clean up repository
ğŸ’¡ Prevention is better than cure: Always use git status before committing and git log --oneline to understand your current state!
